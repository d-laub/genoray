# Symbolic-allele-aware ILEN Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Compute the correct indel length (ILEN) for precise symbolic structural variants (`<DEL>`/`<INS>`/`<DUP>`) on both the VCF and PGEN paths, mark un-sizable symbolic variants with `ILEN = null` + a derived `is_imprecise` expression, and lock the behavior with vcfixture-oracle tests.

**Architecture:** A single private helper builds the corrected per-ALT ILEN polars expression from `ALT`/`REF`/`SVLEN`/`END`/`IMPRECISE` columns (DEL → `-|SVLEN|`, INS/DUP → `+|SVLEN|`, else `null`). The VCF path extracts `SVLEN`/`END` from header-declared INFO via oxbow and persists the corrected ILEN at index-write time; the PGEN path regex-extracts them from the persisted PVAR `INFO` string at index-load time. genoray stays permissive (no drop, no new kwarg); users filter with the existing `filter`/`pl_filter` API (`~is_symbolic` or `~is_imprecise`). Every numpy/numba materialization of ILEN coerces `null → 0`.

**Tech Stack:** Python, Polars (lazy frames, list/string expressions), cyvcf2 (header introspection), oxbow (VCF→DataFrame with `info_fields`), pgenlib/plink2 (PGEN/PVAR), vcfixture 0.6.0 (symbolic fixtures + `GroundTruth` oracle), pytest, Hypothesis, Pixi for the env.

**Spec:** `docs/superpowers/specs/2026-06-04-symbolic-allele-ilen-design.md`

**Run tests with:** `pixi run pytest <path>` (regenerates no data); full suite + data regen: `pixi run test`. The symbolic data fixtures are generated by `tests/data/gen_from_vcf.sh` (invoked by `pixi run test`).

In `genoray/exprs.py`, after the existing `ILEN` definition at the end of the file, add:

```python
# SV type detected from the symbolic ALT token, e.g. "<DEL>" -> "DEL".
_SV_TYPE = pl.element().str.extract(r"^<([A-Za-z0-9:]+)>", 1).str.split(":").list.first()


def symbolic_ilen(
    alt: str = "ALT",
    ref: str = "REF",
    svlen: str = "SVLEN",
    end: str = "END",
    imprecise: str = "IMPRECISE",
) -> pl.Expr:
    """Per-ALT corrected ILEN as ``List[Int32]``.

    Non-symbolic ALTs use the literal ``len(ALT) - len(REF)``. Precise symbolic
    ``<DEL>``/``<INS>``/``<DUP>`` use ``SVLEN`` magnitude (or ``|END - POS|`` for
    ``<DEL>``/``<DUP>`` when ``SVLEN`` is absent): ``-|len|`` for ``<DEL>``,
    ``+|len|`` for ``<INS>``/``<DUP>``. Everything we cannot size precisely
    (``IMPRECISE`` flag, missing length, unsupported type such as ``<BND>``,
    ``<CNV>``, ``<INV>``, ``<*>``) becomes ``null``.

    ``svlen``/``end``/``imprecise`` name scalar columns already extracted by the
    caller (per record). ``SVLEN`` magnitude is read as ``|SVLEN|`` so the VCF
    4.3/4.4 sign-convention flip does not matter.
    """
    ref_len = pl.col(ref).str.len_bytes().cast(pl.Int32)
    svlen_mag = pl.col(svlen).abs().cast(pl.Int32)
    end_mag = (pl.col(end) - pl.col("POS")).abs().cast(pl.Int32)
    mag = pl.coalesce(svlen_mag, end_mag)  # prefer SVLEN, fall back to END

    sv_type = _SV_TYPE
    is_sym = pl.element().str.starts_with("<")
    imprecise_flag = pl.col(imprecise).fill_null(False)

    literal = pl.element().str.len_bytes().cast(pl.Int32) - ref_len

    sized = (
        pl.when(sv_type == "DEL")
        .then(-mag)
        .when((sv_type == "INS") | (sv_type == "DUP"))
        .then(mag)
        .otherwise(None)  # unsupported symbolic type
    )
    # null out anything imprecise or missing a magnitude
    sized = pl.when(imprecise_flag | mag.is_null()).then(None).otherwise(sized)

    return pl.col(alt).list.eval(
        pl.when(is_sym).then(sized).otherwise(literal).cast(pl.Int32)
    )


is_imprecise = pl.col("ILEN").list.eval(pl.element().is_null()).list.any()
"""True if any ALT allele's ILEN could not be precisely determined (an un-sizable
symbolic allele — ``IMPRECISE``, missing ``SVLEN``/``END``, or an unsupported
symbolic type). Such alleles carry ``null`` ILEN. Filter them out with
``pl_filter=~genoray.exprs.is_imprecise`` to keep precise structural variants while
dropping the rest; use ``~genoray.exprs.is_symbolic`` to drop *all* symbolic alleles
(required for haplotype consumers such as genvarloader, which cannot expand any
symbolic ALT)."""
```

> Note: inside `list.eval`, `pl.col(svlen)`/`pl.col(end)`/`pl.col(imprecise)`/`pl.col("POS")` reference outer per-record scalar columns and broadcast across the ALT list. `_SV_TYPE` uses `pl.element()` (the ALT token).

```python
def expected_ilen(truth: GroundTruth, idx: Index) -> list[list[int | None]]:
    """Per-record list of per-ALT expected ILEN from the vcfixture oracle.

    DEL -> -|svlen|, INS/DUP -> +|svlen|, else (unsupported type or no svlen)
    -> None. Mirrors genoray.exprs.symbolic_ilen for the symbolic case. For
    literal ALTs, ILEN is len(ALT) - len(REF).
    """
    n_rec = len(truth.pos)
    rec_ids = list(range(n_rec))[idx] if isinstance(idx, slice) else list(idx)
    out: list[list[int | None]] = []
    for ri in rec_ids:
        ref = truth.ref[ri]
        row: list[int | None] = []
        for at in truth.alts_truth[ri]:
            if at.is_sequence:
                row.append(len(at.alt) - len(ref))  # literal
            elif at.sv_type == "DEL" and at.svlen is not None:
                row.append(-abs(int(at.svlen)))
            elif at.sv_type in ("INS", "DUP") and at.svlen is not None:
                row.append(abs(int(at.svlen)))
            else:
                row.append(None)
        out.append(row)
    return out
```

If the literal sequence is exposed under a different name than `.alt`, adjust (e.g. `at.seq`); the precise-SV branch does not depend on it.
