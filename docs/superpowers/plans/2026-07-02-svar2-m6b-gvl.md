# SVAR2 M6b (gvl side) — Additive Two-Source Reconstruction Kernel Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an SVAR2 haplotype-reconstruction / track-realign path to GenVarLoader that consumes genoray's raw two-channel query result and decodes `var_key ⋈ dense` inline via `svar2-codec` — no intermediate variant table, no per-hap allele copies — **alongside** the retained SVAR 1.0 path.

**Architecture:** gvl's reconstruction inner kernel (`src/reconstruct/mod.rs::reconstruct_haplotype_from_sparse`) is refactored to read per-variant `(v_pos, v_diff, allele)` through a **variant-source closure** (returning `Cow<[u8]>` for the allele, zero-copy on the existing path). The existing global-table indexing becomes one thin source (SVAR 1.0, unchanged behavior); a new **SVAR2 source** merges the per-hap `var_key`+`dense` channels and decodes each key with `svar2-codec::decode_key`. Track realign gets the analogous SVAR2 driver (ilen only). New `#[pyfunction]`s in `src/ffi/mod.rs` take the two-channel numpy arrays; a new additive Python dispatch branch feeds them from genoray's `SparseVar2.overlap_batch`.

**Tech Stack:** Rust (gvl: pyo3 0.28, numpy 0.28, ndarray, rayon; + `svar2-codec` via path-dep), Python 3.10, genoray `svar-2` (editable), seqpro 0.20, pixi/maturin.

## Kernel design (read before Task 3)

The two-source kernel reuses gvl's reconstruction math verbatim; only the per-variant data
*source* changes. In `reconstruct_haplotype_from_sparse`, the loop body touches per-variant
data at exactly three points (`src/reconstruct/mod.rs:87-96`):

```rust
let variant = v_idxs[v] as usize;
let v_pos   = v_starts[variant] as i64;
let v_diff  = ilens[variant] as i64;
let allele_full = &alt_flat[alt_offsets[variant]..alt_offsets[variant+1]];
```

Refactor these behind a source that yields `(v_pos, v_diff, Cow<[u8]>)` for `v in 0..n`:

- **Global-table source (SVAR 1.0, zero-copy):** `Cow::Borrowed(&alt_flat[..])`, `v_pos`/`v_diff`
  from the global arrays via `v_idxs[v]`. `annot_v_idxs` writes the global `variant` id.
- **SVAR2 two-source (new):** per hap, a pre-merged `Vec<(u32 pos, u32 key)>` (var_key ⋈ dense,
  position-sorted); `decode_key(key)` gives `(v_diff, allele)` — `Cow::Owned` for inline SNP/INS,
  `Cow::Borrowed(&lut_bytes[..])` for `Lookup`, empty for `PureDel`. `annot_v_idxs` writes the
  per-hap sequential index (SVAR2 has no global id — consistent with M6c dropping `var_ranges`).

The merge is small (u32 pairs); alleles decode into a reused scratch/`Cow` per variant — **no
per-hap allele table**. If the closure refactor proves too invasive for the fused/spliced
variants, the fallback is a duplicated SVAR2 reconstruction loop (same math, new function) —
decide during Task 3, keeping SVAR 1.0 tests green either way.

//! SVAR2 two-source variant provider: merge a hap's var_key ⋈ dense channels and
//! decode keys via svar2-codec, feeding the reconstruction kernel with no
//! intermediate variant table. Additive to the SVAR 1.0 global-table path.

use std::borrow::Cow;

use svar2_codec::{decode_key, DecodedKey};

/// Decode one uniform key into `(v_diff, allele)`, resolving long-INS via the LUT
/// arrays. Mirrors genoray's `decode_keyref`.
pub fn decode_alt<'a>(key: u32, lut_bytes: &'a [u8], lut_off: &[i64]) -> (i64, Cow<'a, [u8]>) {
    match decode_key(key) {
        DecodedKey::Inline { alt } => (alt.len() as i64 - 1, Cow::Owned(alt)),
        DecodedKey::PureDel { ilen } => (ilen as i64, Cow::Borrowed(&[][..])),
        DecodedKey::Lookup { row } => {
            let s = lut_off[row as usize] as usize;
            let e = lut_off[row as usize + 1] as usize;
            let alt = &lut_bytes[s..e];
            (alt.len() as i64 - 1, Cow::Borrowed(alt))
        }
    }
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_decode_inline_del_lookup() {
        // Pure DEL of length 2 → v_diff -2, empty allele.
        let del = svar2_codec::encode_pure_del(-2);
        let (d, a) = decode_alt(del, &[], &[0]);
        assert_eq!(d, -2);
        assert!(a.is_empty());

        // Lookup row 0 → "ACGT" from the LUT (v_diff = len-1 = 3).
        let lut = b"ACGT".to_vec();
        let off = vec![0i64, 4];
        let lk = svar2_codec::encode_lookup(0);
        let (d, a) = decode_alt(lk, &lut, &off);
        assert_eq!(d, 3);
        assert_eq!(a.as_ref(), b"ACGT");
    }
}
```

/// Merge one hap's `var_key` slice with its carried `dense` set-bits into a single
/// position-sorted `(pos, key)` list (stable: var_key before dense on ties, matching
/// genoray's merge). `dense` is region `r`'s `[ds, de)` window; `present` are this hap's
/// LSB-first presence bits over that window.
#[allow(clippy::too_many_arguments)]
pub fn merge_hap(
    vk_pos: &[i32],
    vk_key: &[i32],
    vk_lo: usize,
    vk_hi: usize,
    dense_pos: &[i32],
    dense_key: &[i32],
    ds: usize,
    de: usize,
    present_bit: impl Fn(usize) -> bool, // present_bit(k) for k in 0..(de-ds)
) -> Vec<(u32, u32)> {
    let mut a: Vec<(u32, u32)> = (vk_lo..vk_hi)
        .map(|i| (vk_pos[i] as u32, vk_key[i] as u32))
        .collect();
    for (k, j) in (ds..de).enumerate() {
        if present_bit(k) {
            a.push((dense_pos[j] as u32, dense_key[j] as u32));
        }
    }
    a.sort_by_key(|&(p, _)| p); // stable; var_key pushed first keeps it ahead on ties
    a
}
```
